# Типы данных полей в Airtable

При создании полей в таблице Airtable используйте следующие типы данных:

## Обязательные поля для таблицы Requests

### 1. **Request ID**
- **Тип поля**: Single line text
- **Описание**: Уникальный идентификатор записи (автоматически заполняется после создания записи)
- **Формат**: Строка (например: "recOjSCdrZHcJQJBF")

### 2. **Created Date**
- **Тип поля**: Date
- **Описание**: Дата создания записи
- **Формат**: Date (только дата, без времени)
- **Формат значения**: `YYYY-MM-DD` (например: "2025-12-14")
- **Важно**: Поле должно быть типа Date, не Date with time

### 3. **Generation Time (seconds)**
- **Тип поля**: Number
- **Описание**: Время генерации запроса в секундах
- **Формат**: Number (целое число)
- **Пример значения**: 45 (секунд)

## Существующие поля (для справки)

- **Feature Description**: Long text
- **KB Links**: Single line text
- **Language**: Single line text
- **Include Questions**: Checkbox
- **Include Acceptance Criteria**: Checkbox
- **Include Test Cases**: Checkbox
- **Include Negative Scenarios**: Checkbox
- **Include Out Of Scope**: Checkbox
- **Use Knowledge Base**: Checkbox
- **Mode**: Single line text
- **Status**: Single line text (значения: "in_progress", "done", "failed")
- **Result Markdown**: Long text

## Обязательные поля для таблицы Feedback

### 1. **Rating**
- **Тип поля**: Number
- **Описание**: Оценка от пользователя (1-5)
- **Формат**: Number (целое число от 1 до 5)
- **Обязательное**: Да

### 2. **Comment**
- **Тип поля**: Long text
- **Описание**: Текст комментария от пользователя
- **Формат**: Текст (без изображений)
- **Обязательное**: Нет

### 3. **Email**
- **Тип поля**: Email или Single line text
- **Описание**: Email адрес пользователя (необязательное поле)
- **Формат**: Email адрес (валидируется на фронте и бэкенде)
- **Обязательное**: Нет

### 4. **Request** (опционально)
- **Тип поля**: Link to another record
- **Описание**: Связь с записью из таблицы Requests
- **Формат**: Связь с таблицей Requests
- **Обязательное**: Нет

## Примечания

- Все поля, кроме новых, уже должны существовать в таблице
- Если поле "Created Date" уже существует как "Date with time", его нужно переименовать или создать новое поле типа "Date" (без времени)
- Поле "Request ID" можно сделать уникальным, если нужно предотвратить дубликаты
- Уникальный ID записи в таблице Feedback генерируется автоматически Airtable
- В поле Comment запрещена вставка изображений (проверяется на фронте)
